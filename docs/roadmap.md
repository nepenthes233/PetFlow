# PetFlow 开发路线图

本文档只描述后续开发的大方向，不把任务拆到很细。当前仓库已经可以作为
`main` 分支开发基线，具体架构边界见 `docs/architecture.md`，协作规范见
`docs/development.md`。

## 1. 当前基线

当前项目已经具备一个可运行的基础版本：

- Python 3.12 + Tkinter + Canvas 桌面应用。
- 任务图数据模型、JSON 保存加载、样例图入口。
- 节点和边的创建、编辑、删除、拖拽和状态切换。
- Dependency 成环检测，Routine / Recommendation / Trigger 边类型支持。
- 基础布局、缩放、平移和重置视图。
- 节点详情字段、资源节点、附件入口、剪贴板捕获。
- 本地推荐引擎和推荐理由。
- 图内桌宠、气泡和轻量移动动画。
- Agent mock / API 双路径、任务图生成、节点拆分和预览。
- Review、Focus Mode 等演示入口。
- 核心单元测试覆盖主要服务和规则。

后续开发不需要重搭框架，重点是把已有能力打磨成稳定、可演示、可协作扩展的版本。

## 2. 开发主线

后续开发按三条主线推进：

```text
Todo 图能力升级 -> Agent 功能完善 -> 桌宠体验增强
```

这三条主线不是完全割裂的。Todo 图是基础，Agent 需要依赖稳定的图模型和服务层，
桌宠则负责把 Todo 图和 Agent 的状态反馈给用户。

## 3. 主线一：Todo 图能力升级

目标是让 PetFlow 先成为一个可靠的任务图工具。

主要方向：

- 打磨 Canvas 图编辑体验，让节点和边的创建、选择、拖动、删除更顺手。
- 完善节点详情，让 Task、Routine、Resource、Checkpoint、Reward 的字段更可见、可编辑、可保存。
- 完善 Resource、附件、Checklist、标签、实际耗时等工作流细节。
- 强化 Routine 的到期、过期、完成刷新和展示逻辑。
- 让推荐结果和推荐理由更稳定、更容易理解。
- 保持 JSON 保存加载兼容，持续维护可演示的 `sample_graph.json`。

这条主线优先级最高。图能力稳定后，Agent 和桌宠的效果才有基础。

## 4. 主线二：Agent 功能完善

目标是让 Agent 成为任务图的辅助规划工具，而不是直接替代用户操作。

主要方向：

- 完善自然语言生成任务图：输入目标，生成结构化节点和边。
- 完善节点拆分：根据当前节点、前置依赖、后续节点和上下文生成子任务。
- 强化 Agent 结果预览，让用户在应用前能看清会新增什么。
- 强化 Agent 校验，避免异常 JSON、重复 ID、非法边、Dependency 成环等问题破坏当前图。
- 保留 mock 模式，保证没有 API Key 或网络不可用时仍能演示。
- 扩展 Review 能力，先用本地统计生成复盘，再按需接入 Agent 改写。

Agent 的基本原则是：先预览，后确认，再通过 `GraphService` 写入图。

## 5. 主线三：桌宠体验增强

目标是让桌宠从图上的装饰变成任务状态反馈入口。

主要方向：

- 梳理桌宠状态，例如 idle、think、happy、angry、sleep、move。
- 让桌宠响应推荐、任务完成、Routine 到期、Agent 工作和 Focus Mode 等事件。
- 打磨气泡文本、颜色、位置和轻量移动动画。
- 保持无额外图片资源时也能演示，后续再考虑 PNG/GIF 等表现增强。
- Focus Mode 先保证降级可用，再逐步增强前台窗口检测和偏离提醒。

桌宠的重点不是复杂动画，而是让用户能直观看到“现在该做什么”和“系统为什么提示”。

## 6. 协作节奏

建议近期先把 `main` 分支作为稳定基线提交，然后按功能分支推进：

- 图能力相关改动优先合并，避免 Agent 和桌宠建立在不稳定接口上。
- 每个分支只处理一个清晰主题。
- 改动图数据结构时，必须同步测试和文档。
- UI 改动要做基本手动验证。
- Agent 和系统能力必须保留 mock 或降级路径。

推荐的阶段节奏：

```text
第一阶段：Todo 图打磨和样例图完善
第二阶段：Agent 预览、校验和应用流程完善
第三阶段：桌宠状态反馈和演示体验增强
第四阶段：最终演示、报告和文档收口
```

## 7. 最终目标

最终版本应能稳定演示以下闭环：

1. 打开程序并加载样例图。
2. 创建、编辑、拖动、删除节点和边。
3. 展示 Dependency 成环拒绝和 Routine 环允许。
4. 编辑节点详情、资源、附件和 Checklist。
5. 标记任务完成，得到下一步推荐和推荐理由。
6. 桌宠移动到推荐节点并显示气泡。
7. 使用 Agent 生成任务图或拆分节点。
8. 使用 Review、剪贴板捕获或 Focus Mode 作为增强演示。
9. 保存、关闭、重新加载后数据保持一致。

这个路线图会随着开发进展继续调整，但不再把早期 Qt 方案作为当前实现方向。
